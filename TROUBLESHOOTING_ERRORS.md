Ce guide répertorie les codes d'erreur standards de l'API Documenso pour
faciliter le diagnostic des intégrations.

| Code d'Erreur             | Signification                                                             | Solution Recommandée                                                                                                                                                                |
| :------------------------ | :------------------------------------------------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ALREADY_EXISTS`          | La ressource que vous tentez de créer existe déjà.                        | Vérifiez si l'élément (utilisateur, document, webhook) n'a pas déjà été instancié. Modifiez votre requête pour effectuer une mise à jour (PUT/PATCH) au lieu d'une création (POST). |
| `EXPIRED_CODE`            | Le code d'accès ou le jeton fourni a expiré.                              | Générez un nouveau code d'accès ou demandez un nouveau lien d'invitation avant de relancer la requête.                                                                              |
| `INVALID_BODY`            | Le corps de la requête (payload) est mal formaté.                         | Vérifiez la structure de votre payload JSON. Assurez-vous qu'il respecte strictement la syntaxe attendue et qu'aucun champ obligatoire ne manque.                                   |
| `INVALID_REQUEST`         | La requête globale est mal formée ou invalide.                            | Inspectez les paramètres de votre appel API (URL, query parameters, headers). Corrigez la syntaxe de la requête.                                                                    |
| `RECIPIENT_EXPIRED`       | Le lien de signature ou l'accès du destinataire a expiré.                 | Générez et renvoyez une nouvelle invitation au signataire concerné.                                                                                                                 |
| `LIMIT_EXCEEDED`          | Le quota d'utilisation de votre compte a été dépassé.                     | Vérifiez les limites de votre forfait actuel. Mettez à niveau votre abonnement ou patientez jusqu'au renouvellement de votre cycle de facturation.                                  |
| `NOT_FOUND`               | La ressource demandée est introuvable (Erreur 404).                       | Vérifiez l'identifiant (ID) de la ressource (enveloppe, document, webhook) transmis dans l'URL. Assurez-vous qu'elle n'a pas été supprimée.                                         |
| `NOT_IMPLEMENTED`         | La fonctionnalité demandée n'est pas encore supportée par le serveur.     | Consultez la documentation pour vérifier les méthodes disponibles. Ne tentez pas d'utiliser cet endpoint pour le moment.                                                            |
| `NOT_SETUP`               | La configuration requise pour cette action est incomplète.                | Accédez aux paramètres de votre compte ou de votre intégration et finalisez la configuration avant de réessayer.                                                                    |
| `INVALID_CAPTCHA`         | La validation du jeton de sécurité (Captcha) a échoué.                    | Assurez-vous que le jeton Captcha est correctement généré côté client et transmis sans altération dans votre requête.                                                               |
| `UNAUTHORIZED`            | Authentification manquante ou invalide (Erreur 401).                      | Vérifiez que votre clé API est correcte, active, et correctement formatée dans l'en-tête `Authorization` (ex: `Bearer <VOTRE_CLE>`).                                                |
| `FORBIDDEN`               | Accès refusé à la ressource (Erreur 403).                                 | Vérifiez que votre clé API ou votre compte utilisateur dispose des permissions et des rôles nécessaires pour exécuter cette action spécifique.                                      |
| `UNKNOWN_ERROR`           | Erreur interne inattendue du serveur (Erreur 500).                        | Réessayez ultérieurement. Si le problème persiste, contactez le support technique en fournissant le payload de votre requête et l'heure de l'incident.                              |
| `RETRY_EXCEPTION`         | L'opération a échoué temporairement mais peut être relancée.              | Implémentez une logique de réessai automatique (retry) dans votre code, idéalement avec un délai exponentiel (exponential backoff).                                                 |
| `SCHEMA_FAILED`           | Échec de la validation stricte du schéma de données.                      | Vérifiez que les types de données envoyés (string, number, boolean) correspondent exactement au schéma de l'API.                                                                    |
| `TOO_MANY_REQUESTS`       | Limite de requêtes atteinte (Rate limiting - Erreur 429).                 | Ralentissez la fréquence de vos appels API. Implémentez une gestion d'attente basée sur les en-têtes de limitation (Rate-Limit headers).                                            |
| `TWO_FACTOR_AUTH_FAILED`  | L'authentification à deux facteurs (2FA) a échoué.                        | Vérifiez le code 2FA fourni. Assurez-vous qu'il a été saisi correctement et qu'il n'a pas expiré.                                                                                   |
| `WEBHOOK_INVALID_REQUEST` | La requête liée au webhook est invalide.                                  | Vérifiez la configuration de votre endpoint de réception. Assurez-vous que l'URL est correcte et que votre serveur accepte les requêtes POST de Documenso.                          |
| `ENVELOPE_DRAFT`          | L'action est impossible car l'enveloppe est encore à l'état de brouillon. | Finalisez la configuration de l'enveloppe et passez-la à l'état "envoyé" avant de tenter cette opération.                                                                           |
| `ENVELOPE_COMPLETED`      | L'action est impossible car l'enveloppe est déjà complétée.               | Aucune modification (ajout de signataire, modification de document) ne peut être apportée à un document dont le processus de signature est terminé.                                 |
| `ENVELOPE_REJECTED`       | L'action est impossible car le document a été refusé par un signataire.   | Le flux de signature est définitivement interrompu. Créez une nouvelle enveloppe si vous souhaitez soumettre à nouveau le document.                                                 |
| `ENVELOPE_LEGACY`         | L'action est impossible car l'enveloppe utilise un format obsolète.       | Cette enveloppe a été créée avec une ancienne version du système. Recréez le document avec la version actuelle de l'API pour interagir avec.                                        |
