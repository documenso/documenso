# Documenso — One-click AWS Deploy

`documenso.yml` is a self-contained AWS CloudFormation template that deploys Documenso (this fork) onto ECS Fargate with RDS PostgreSQL and S3 for document uploads.

## Launch Stack

Click the button below (replace `<owner>` with your GitHub org/user if you've forked this repo):

```
https://console.aws.amazon.com/cloudformation/home#/stacks/create/review?templateURL=https://raw.githubusercontent.com/<owner>/documenso/main/cfn/documenso.yml&stackName=documenso
```

## What You'll Be Asked

The CloudFormation console groups parameters into sections:

- **Environment** — `EnvName` (used in resource naming)
- **Networking & DNS** — pick a VPC, 2+ public subnets, a domain, and a Route 53 hosted zone that already owns the apex
- **Application** — container image URI, app name, allowed signup domains, SMTP "From" address
- **SSO (Microsoft)** — tenant ID (use `common` for multi-tenant)
- **Sizing** — RDS/Fargate sizing (defaults: `t4g.small`, 512 CPU / 1024 MiB, 1 task)

## After Deploy (required)

The template creates an `app-config` secret with placeholder values. Before Documenso will sign in users, update these keys via Secrets Manager (`documenso/<env>/app-config`):

- `NEXT_PRIVATE_ENCRYPTION_KEY` and `NEXT_PRIVATE_ENCRYPTION_SECONDARY_KEY` (32+ random chars each)
- `NEXT_PRIVATE_MICROSOFT_CLIENT_ID` / `_CLIENT_SECRET` (from your Entra app registration)
- `NEXT_PRIVATE_SMTP_HOST` / `_PORT` / `_USERNAME` / `_PASSWORD` (SES SMTP credentials)
- `NEXT_PRIVATE_SIGNING_PASSPHRASE` and `NEXT_PRIVATE_SIGNING_LOCAL_FILE_CONTENTS` (base64 of your `.p12` cert)
- Update `documenso/<env>/database-url` with the RDS endpoint from the stack outputs

The repo's `.claude/skills/deploy-to-aws` skill walks you through all of this interactively.

## Prefer CDK?

The same template is generated by the CDK in `infra/`:

```bash
cd infra && npm install
npx cdk synth -a "npx ts-node bin/cfn-synth.ts"
```

For CI/CD-driven deploys (internal mode), set the GitHub repo variables documented in `.github/workflows/deploy-aws.yml` and push to `main`.
